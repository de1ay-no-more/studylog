使用 localStorage 实现前端数据持久化

2026年5月18日·JavaScript·预计阅读时间
JavaScriptlocalStorage表单验证

没有后端时，浏览器端仍然可以使用 localStorage 保存少量数据。本项目的留言板就是一个典型示例：用户提交昵称和留言后，即使刷新页面，留言仍然可见。

localStorage 的基本用法

localStorage.setItem("key", "value");
const value = localStorage.getItem("key");

因为 localStorage 只能保存字符串，所以数组或对象需要先用 JSON.stringify 转换，读取时再用 JSON.parse 还原。

留言板实现思路

  1. 页面加载时读取本地留言数组。
  2. 提交表单前检查昵称和留言是否为空、长度是否合理。
  3. 通过 unshift 把新留言放到数组最前面。
  4. 保存到 localStorage 并重新渲染留言列表。

安全渲染

用户输入不能直接拼接为 HTML，否则可能出现脚本注入问题。这个项目中使用 escapeHTML() 将特殊字符转义后再显示。

localStorage 很适合课程项目中的轻量交互，但真实应用仍应由后端完成身份验证、数据校验和长期存储。
